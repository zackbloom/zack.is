You Might Not Need jQuery.ready()

Over the years it has become something of a mantra amoung JavaScript developers that files are inevitably wrapped in a call to $():

$(function(){

  // Real code here
  
})

Many of these developers may not even know that $(function(){ }) is a shorthand for a call to jQuery.ready. It is very often not required.

If you are including your code at the end of the body, you don’t need it.

Script tags are able to access any element defined before them in the HTML file. This means if your script tag is at the end of the body, your code can already access any HTML element you need.

If you are not interacting elements in the body, you don’t need it

Installing an analytics tracking script? Just append it to document.head.

If you want your new elements to render without a flicker, don’t use it.

The point where $.ready runs is often after the page has initially been rendered. If you’re interested in doing something to manipulate the page, one option is to use Mutation Observers. Another is to not make that particular change with JavaScript.

What To Do Instead

Just wrap your code with a plain function! It will run immediately, but give you the same variable scoping:

(function(){

  // Real code here

})();

For a deeper dive into when code runs, see How To Decide When Your Code Should Run.
